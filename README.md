# 🐾 Poe 2 — Vorana's Saga Rumor Tracker

## Ня! Программа для подсчёта уникальных руморов при работе с Vorana's Saga (=^･ω･^)

Windows-программа, которая через оверлей показывает количество уникальных "rumors" при работе с предметом **Vorana's Saga**. Работает через OCR и template matching.

---

## 📋 Требования

| Компонент | Версия | Примечание |
|-----------|--------|------------|
| Python | 3.9+ | Обязательно галка "Add to PATH" при установке! |
| Windows | 10/11 | Trello OCR требует Windows |
| Path of Exile 2 | Запущена | Процесс `PathOfExile2.exe` должен работать |

## 🚀 Быстрый старт (3 шага)

### Шаг 1: Установи Python

Если ещё не установлен — скачай с [python.org](https://www.python.org/downloads/) и **обязательно** поставь галочку **"Add Python to PATH"** при установке! Затем перезапусти терминал.

Проверь установку:
```bash
python --version
# Должно показать: Python 3.x.x
```

### Шаг 2: Скачай проект

Открой командную строку и клонируй репозиторий:
```bash
git clone https://github.com/BrondIl/hermes.git
cd hermes
```

> **Или** скачай ZIP с GitHub, распакуй в любое место.

### Шаг 3: Запусти скрипт

Открой `run.bat` двойным кликом или из терминала:
```bash
# Из папки проекта:
.\run.bat
```

Скрипт автматически:
1. Создаёт виртуальное окружение (`venv`) если нет
2. Устанавливает зависимости (`requirements.txt`)
3. Запускает программу

## 🎮 Как использовать в игре

### Автоматический режим (рекомендуется)

При первом запуске программа **сама** найдёт все нужные элементы PoE2 UI — **ни рисовать, ни не двигать зоны вручную не нужно**!

Как работает авто-детекция:
1. 📸 Программа делает скриншот всей рабочей области экрана
2. 🔍 **OCR (PaddleOCR)** ищет текст в правой верхней части экрана → находит таблицу руморов, вычисляет её bounding box
3. 🎨 **Цветовой анализ** ищет золотистые предметы во вкладке инвентаря → определяет положение Vorana's Saga
4. ✅ Все найденные координаты применяются в runtime
5. ⚠️ Если OCR недоступен — используется **fallback из ROI % в config.py**

После обнаружения:
- Счётчик "Слухи: X" отображается поверх игры над таблицей руморов
- При наведении мыши в зону слева от таблицы руморов начинается трекинг
- Кликни F9 для start/stop

### Если авто-обнаружение не работает


```python
# ROI: область экрана где появляются слухи (в процентах от разрешения)
class ROI:
    X1 = 0.78  # слева (0.0 = левый край, 1.0 = правый)
    Y1 = 0.05  # сверху
    X2 = 0.97  # справа
    Y2 = 0.35  # снизу
```

### Настройки оверлея

В `config.py` можно настроить внешний вид счётчика:

```python
class Overlay:
    COUNTER_X     = 10       # X позиция оверлея
    COUNTER_Y     = 10       # Y позиция
    COLOR_IDLE    = (0, 255, 255)   # Циан — ждём
    COLOR_ACTIVE  = (0, 255, 0)     # Зелёный — активна...
```

## 🐛 Решение проблем

**Программа не запускается — ошибка `ModuleNotFoundError`:**
```bash
pip install -r requirements.txt
```

**Ошибка Tesseract — OCR не работает:**
1. Скачай [Tesseract-OCR для Windows](https://github.com/UB-Mannheim/tesseract/wiki)
2. Установи в обычное место (например, `C:\Program Files\Tesseract-OCR\tesseract.exe`)
3. Перезагрузи компьютер

**Руморы не отслеживаются — координаты неверные:**
1. Открой игру на полный экран или окно
2. Запусти программу с флагом debug: `python main.py`
3. Программа создаст `debug_screenshot.png` в папке проекта
4. Проверь что красный маркер находится в зоне где руморы появляются
5. Настрой `config.py` → класс `ROI`
